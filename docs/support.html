Quick Troubleshooting

The display is blank or shows "Ready to pair"

The display has no saved configuration. Open the GitGrid app, connect to the display via Bluetooth, and follow the setup steps to configure WiFi and your GitHub username.

The app can't find the display

Make sure Bluetooth is enabled on your phone and you're within a few feet of the display. If the display is already configured and showing a grid, it's still discoverable — just open the app and it will auto-connect.

WiFi won't connect

GitGrid only supports 2.4 GHz WiFi networks. Most routers broadcast both 2.4 GHz and 5 GHz — make sure you're connecting to the 2.4 GHz network. Double-check your password and try the "Enter manually" option if your network doesn't appear in the scan.

Display says "No data / retrying"

The display is connected to WiFi but couldn't fetch your GitHub data. It will automatically retry with increasing intervals. Check that your internet connection is working. If the problem persists, try power cycling the display.

My contributions aren't showing

Make sure your GitHub contributions are set to public in your GitHub profile settings. Also verify that your username is spelled correctly (case-sensitive). If you have private contributions, you can add a custom Personal Access Token in the app under Home > GitHub Token.

How do I factory reset?

From the app: go to Home > Device > Factory Reset Device. This erases all settings and returns the display to pairing mode. If you can't connect via the app, you can perform a hardware reset — see the Help & FAQ section in the app for detailed instructions.

How do I update the firmware?

GitGrid checks for firmware updates automatically every 6 hours. You can also trigger a manual check from the app under Home > Device > Check for Update. Updates download and install automatically — the display will briefly show "updating firmware" and reboot.
